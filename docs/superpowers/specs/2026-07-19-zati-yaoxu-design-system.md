# ZATI「曜序」设计系统

> 状态：已批准  
> 日期：2026-07-19  
> 适用范围：ZATI MVP 1.0 及后续产品界面的视觉与交互基础

## 1. 设计决策

ZATI 采用：

> **Material 3 规则 × ZATI 自有组件**

Material 3 负责不直接形成品牌识别的基础质量：语义令牌、组件状态、焦点、键盘操作、响应式、动效原则和无障碍要求。

ZATI「曜序」负责用户能够感知的产品身份：颜色取舍、字体、轮廓、人格牌、命核、星域、星轨、正逆位表达、报告布局和分享资产。

页面与业务代码只使用 ZATI 组件 API，不暴露 Material 命名。底层实现可以替换，品牌和页面合同保持稳定。

## 2. 设计目标

系统需要同时满足：

1. 第一眼是可信、当代的数字人格产品，不是算命网站。
2. 测试过程安静、清楚、低认知负担。
3. 结果揭晓具有身份感和克制的东方仪式感。
4. 报告适合长时间阅读，分享资产具有鲜明辨识度。
5. 深色主体验、浅色报告、移动端与桌面端共享同一套语义合同。

设计原则：

- Material 负责看不见的正确性，ZATI 负责看得见的独特性。
- 神秘感来自文字、构图、轨迹和节奏，不来自古风装饰堆叠。
- 只在结果揭晓这一处集中使用强仪式感，其余界面保持克制。
- 正位、逆位、四化和星曜不通过颜色制造吉凶或人格高低。

## 3. 系统架构

### 3.1 基础行为层

继承 Material 3 的交互合同，但不继承默认外观：

- Button
- Icon button
- Dialog
- Text field
- Radio
- Checkbox
- Progress
- Menu
- Tooltip

基础行为层负责：焦点、键盘操作、禁用、加载、选中、错误、触控区和语义属性。

### 3.2 组合体验层

ZATI Shell 将基础控件组合成业务模式：

- App header
- Mode selector
- Question step
- Answer scale
- Axis meter
- Report section
- Share sheet
- Birth form

### 3.3 品牌核心层

ZATI Signature 是不可被通用组件替代的产品资产：

- 本命人格牌
- 命核印记
- 星域标记
- 星轨生成仪式
- 正逆位对照
- 星曜四原型
- 人生季节
- 分享卡

## 4. 色彩系统

ZATI 不启用 Material 动态取色。品牌颜色固定映射到 Material 3 语义角色，确保视觉和文化表达稳定。

| 令牌 | 色值 | 语义角色 | 使用范围 |
| --- | --- | --- | --- |
| 曜黑 | `#0B0B10` | Background | 深色主背景 |
| 月白 | `#F2F0EA` | On background | 主文字、浅色报告背景 |
| 星金 | `#C5A66A` | Primary | 单一主行动、进度、结果代码、选中态 |
| 玄紫 | `#58466F` | Secondary | 辅助行动、命盘轨迹、解释信息 |
| 朱砂 | `#A94F43` | Error / Tension | 错误、风险、逆位张力，不表示厄运 |
| 青玉 | `#557D70` | Success / Growth | 完成、恢复、归位、成长，不表示吉星 |
| 玄面 | `#17161C` | Surface | 控件与局部表面 |
| 星界 | `#393640` | Outline | 分隔线、控件描边、边界 |

规则：

- 一个页面最多出现一个星金主行动。
- 玄紫只用于解释层和次级状态，不能成为满屏主色。
- 朱砂和青玉必须同时配合图标或文字，颜色不能是唯一信号。
- 浅色主题用于长图、PDF、打印和必要的无障碍场景，不是独立品牌风格。
- 实现阶段应使用 Material Color Utilities 校验色调和对比，但不允许用户动态换色。

## 5. 字体系统

| 角色 | 字体 | 建议字号 | 用途 |
| --- | --- | --- | --- |
| Display | Noto Serif SC | 40–56px | 首页主标题、结果揭晓 |
| Title | Noto Serif SC | 24–32px | 人格牌名、报告章节标题 |
| Body | Noto Sans SC | 15–18px | 题目、报告正文、说明 |
| Utility | IBM Plex Mono | 11–14px | 四字母代码、题号、数据标签 |

规则：

- 人格牌名和仪式型标题使用宋体气质，正文保持现代无衬线可读性。
- 正文字号在移动端不得低于 15px。
- 正文最大行宽为 `68ch`。
- 不使用随视口宽度连续缩放的字体。
- 字间距保持 `0`，不使用负字间距营造紧凑感。

## 6. 间距、尺寸与轮廓

- 基础网格：4px。
- 默认间距步进：4、8、12、16、24、32、48、64px。
- 最小触控区：48×48px。
- 输入框、答题选项圆角：2px。
- 面板与命令按钮圆角：4px。
- 弹层与工具面板圆角：8px。
- Pill 仅用于状态标签、过滤条件和短代码，不用于普通按钮。
- 人格牌使用固定 `2:3` 比例、独立边框和角部细节，不套用通用 Card。

表面规则：

- 长报告使用留白、分隔线和全宽色带组织，不把章节做成浮动卡片。
- 禁止卡片嵌套卡片。
- 默认层级依靠色面与描边，不依赖阴影。
- 只有 Dialog、Menu、Tooltip 等浮层使用柔和阴影。

## 7. 核心组件合同

### 7.1 命令组件

- 一个视图只设置一个主行动。
- 主按钮使用星金实色；次级按钮使用描边或低强调文字。
- 图标按钮使用熟悉图标并提供可访问名称和 Tooltip。
- 加载状态不得改变按钮尺寸。

### 7.2 测试组件

- 每页一题，题目维度不对用户显示。
- 整行选项可点击，触控区不低于 48px。
- 选中态同时使用描边、底色和符号，不只改变颜色。
- 进度条尺寸稳定，不因题号或反馈文案变化而跳动。
- 退出、返回和恢复进度具有明确确认和恢复路径。
- 六级倾向控件在移动端必须可单手操作并清楚标示两端语义。

### 7.3 报告组件

- 结果首屏先交付身份：代码、牌名、命核、牌辞和人间映射。
- 四维比例同时显示数值和图形。
- 45–55 的灵活交界区显示文字说明，不能只使用弱化色。
- 免费、注册和付费区块通过内容边界与行动提示区分，不用模糊遮罩诱导。
- 工作、关系和成长内容使用章节带与分隔线，不使用重复 Card 网格。

### 7.4 状态组件

以下状态必须具有明确说明和下一步：

- 加载和计算中
- 网络或提交失败
- 测试未完成
- 缺少出生信息
- 出生时间准确度低
- 灵活交界区
- 内容尚未解锁
- 报告生成降级

## 8. 页面模式

### 8.1 首页

- 首屏直接呈现 ZATI 品牌与测试主行动，不做营销型卡片拼贴。
- 背景使用克制的命盘轨迹或人格牌真实视觉资产。
- 主文案为“以选择看见自己”，价值说明放在支持文案。
- 首屏必须露出下一段内容，桌面与移动端均不能被满屏 Hero 完全封闭。
- 测试模式作为清晰选项呈现，不把尚未上线的 1.5 功能伪装成可用按钮。

### 8.2 测试页

- 单列、聚焦、稳定尺寸。
- 顶部显示题号、进度和退出操作。
- 题目切换不改变整体容器位置。
- 选项顺序可由题库规则反转，视觉结构保持一致。
- 阶段反馈短暂出现，但不能遮挡下一题或强迫等待。

### 8.3 生成仪式

- 星轨收束、人格牌翻开、身份文字落位构成一次完整揭晓。
- 提供跳过动画操作，但不能跳过实际结果计算。
- 不使用虚假加载延长仪式。
- 结果已经可用时，动画总长不超过 1.4 秒。

### 8.4 结果页

- 移动端首屏完整呈现本命人格牌和身份摘要，并露出下一节。
- 桌面端人格牌与摘要并排，正文保持窄阅读栏。
- 报告章节按照身份、四轴、正逆位、压力归位、工作、关系、成长组织。
- 分享操作保持可见但不抢占主内容。
- 1.5 的星曜、四化和人生季节必须与 1.0 内容明确分层。

## 9. 动效系统

动效只服务三个时刻：

| 类型 | 时长 | 用途 |
| --- | --- | --- |
| 状态确认 | 120–180ms | 选项、按钮、焦点、进度即时反馈 |
| 页面转换 | 220–320ms | 题目切换、章节进入、弹层出现 |
| 本命揭晓 | 900–1400ms | 星轨收束、翻牌、身份文字落位 |

规则：

- 状态动效只改变色面、描边、透明度或不超过 4px 的位移。
- 禁止散落的滚动入场动画和持续漂浮装饰。
- 动态内容不能引起布局尺寸变化。
- `prefers-reduced-motion: reduce` 时关闭轨道旋转和 3D 翻牌，改为 180ms 交叉淡入。
- 降级模式必须保留相同信息、操作顺序和完成时间。

## 10. 响应式系统

### Compact：小于 600px

- 单列、单手测试模式。
- 主行动可停靠底部安全区。
- 题目容器最大宽度 32rem。
- 人格结果首屏完整出现并露出下一节。

### Medium：600–1023px

- 结果牌与身份摘要可并排。
- 报告正文仍保持单列。
- 分享操作可停靠内容右侧。

### Expanded：1024px 及以上

- 左侧可固定报告目录。
- 中央正文不超过 68ch。
- 右侧只放分享操作或关键摘要，不构成第二正文列。
- 测试页继续保持单列聚焦，不扩展为多列问卷。

## 11. 无障碍合同

- 正文与背景达到 WCAG AA；大字至少达到 3:1 对比度。
- 所有交互具有清晰的 `:focus-visible` 状态。
- 全流程可以只用键盘完成。
- 表单具有真实 Label、描述和错误关联。
- 颜色不是状态、正逆位、完成与错误的唯一信号。
- 四轴数值、图表和进度均提供可朗读文本。
- 结果揭晓提供 reduced-motion 和跳过动画。
- 页面缩放至 200% 时不丢失内容或命令。
- 最长牌名、最长错误文案和中英文混排不得溢出容器。
- 出生时间不确定时明确显示置信度、内容降级和原因。

## 12. Material 3 开源资产边界

### 可采用

- Material 3 的颜色角色、排版角色、状态、布局、动效和组件行为规范。
- Material Color Utilities 的 TypeScript 实现，用于调色和对比校验。
- `material-components/material-web` 的状态、无障碍和组件源码作为参考实现。

`material-web` 使用 Apache License 2.0。若直接复制或修改源码，必须保留许可、版权及相关归属声明，并标记修改。

### 不直接采用

- 不将整套 `material-web` 作为 React 页面组件基础。它使用 Web Components，直接依赖会增加 Shadow DOM 定制、事件封装和品牌外形耦合。
- 不将 MUI 默认 Theme 或默认组件外观作为成品视觉。
- 不复制 Material 默认动态色、超大圆角、药丸按钮、彩色面板和阴影层级。

### 推荐实现

- React + TypeScript。
- 使用原生语义元素或可访问的无样式 primitives 承载行为。
- 使用 CSS Custom Properties 表达 ZATI 语义令牌。
- 所有业务组件只依赖 ZATI tokens 和 ZATI component API。
- 组件源码、令牌和业务内容分层存放，避免人格内容与样式耦合。

## 13. 验收与测试

### 视觉回归

- 首页
- 题目页
- 生成仪式
- 结果首屏
- 长报告
- 分享卡

### 固定视口

- 390×844
- 768×1024
- 1440×900

### 交互验收

- 键盘完成整套测试。
- 焦点顺序和返回恢复正确。
- 正常动效与 reduced-motion 双模式一致。
- 加载、错误、空态、未解锁和低置信度状态完整。

### 内容与版式压力测试

- 最长人格牌名。
- 200% 页面缩放。
- 中英文代码混排。
- 长题目、长选项和长错误信息。
- 深色主体验与浅色报告/打印。

### 性能验收

- 首屏不被大型图片或字体阻塞。
- 揭晓动画不造成明显掉帧或布局跳动。
- 分享图生成失败时提供可重试和文本分享降级。

## 14. 实现禁区

- 禁止直接以 MUI 默认 Theme 作为成品视觉。
- 禁止在页面中散落颜色、间距、圆角和阴影魔法值。
- 禁止把每个报告章节包装成圆角浮动卡片。
- 禁止卡片嵌套卡片。
- 禁止用 Canvas 文本替代可选择、可朗读的结果内容。
- 禁止添加不可跳过的虚假加载。
- 禁止把星金、朱砂、青玉解释成吉凶层级。
- 禁止用纯装饰动画持续消耗性能或干扰阅读。
- 禁止让底层 Material 命名泄漏到业务组件 API。

## 15. 设计完成定义

设计系统在满足以下条件时可进入实现：

1. 本文档中的令牌、组件所有权、页面模式和体验合同保持一致。
2. MVP 1.0 的首页、测试、揭晓、结果和分享均能由现有模式覆盖。
3. 1.5 和 2.0 功能可以扩展品牌组件，而不修改基础人格测试合同。
4. Material 3 只作为语义与行为根基，不决定 ZATI 的外观和业务 API。
5. 视觉回归、响应式、键盘、动效降级和内容伦理均有明确验收路径。
